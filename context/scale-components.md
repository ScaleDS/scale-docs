# Scale Design System — Component Reference

> Scale.Design System 3.0.0 — 3,000+ components, 703 variables, 936 icons, 40 content modules.

## Tokens

| Page | Description |
|---|---|
| **Brand** | The Scale and Figma logos across all sizes and styles, plus a set of fictional company logos ready to drop into your designs. |
| **Color** | Primitive color palette and semantic color variables — the full system powering every light and dark mode token across Scale. |
| **Iconography** | 900+ icons across 16, 24, and 32px sizes — based on Feather Icons — with status, social, and pictogram sets, plus guidance on sizing, crafting, and swapping. |
| **Image** | Placeholder component that holds space for an image until real content is dropped in. |
| **Layout and Spacing** | Grid definitions and spacing variables for every breakpoint from mobile (393px) to desktop (1920px), with a full spacing pixel table. |
| **Overlay** | Semi-transparent backdrop components across a range of opacities, used to dim content behind modals, sheets, and dialogs. |
| **Materials and Surfaces** | Liquid Glass effects, surface elevation levels, and frosted backdrop treatments for building depth and hierarchy in app interfaces. |
| **Typography** | Complete type scale for Desktop, Mobile, iOS, and Android — covering Heading, Text, Link, and Label styles with variables that switch automatically between platforms. |

## Components — Global

| Component | Description |
|---|---|
| **Accordion** | Vertically stacked sections that expand and collapse, perfect for keeping dense content tidy and scannable. |
| **Alert** | Inline messages that surface important information, warnings, errors, or success states right where the user needs them. |
| **Avatar** | Circular representations of a person or entity — showing a photo, initials, or a placeholder icon. |
| **Badge** | Status indicators that show notifications, or state on top of another element. |
| **Banner** | Full-width messages pinned to the top of the page for system-wide announcements or important notices. |
| **Breadcrumb** | Trail of linked pages showing users exactly where they are within the site hierarchy. |
| **Button** | The go-to element for triggering actions — tappable, clickable, and hard to miss. |
| **Button Pill** | Like Button, but with fully rounded ends for a softer, more contemporary feel. |
| **Button Icon** | An icon-only button that communicates an action without needing a label. |
| **Button Group** | Related buttons bundled together in a row with consistent spacing and shared styling. |
| **Card** | Self-contained surfaces that group related content and actions around a single subject, easy to scan at a glance. |
| **Card Selector** | Cards that double as selection controls — a richer, more visual alternative to radio buttons or checkboxes. |
| **Checkbox** | Lets users toggle one or more independent options on or off, each behaving independently from the rest. |
| **Checkmark** | Visual confirmation that something is selected, complete, or successful. |
| **Date Picker** | Calendar-based input for selecting a date or range, so users never have to type one out manually. |
| **Divider** | Thin lines that cleanly separate sections or groups of content. |
| **File Upload** | Lets users select and upload files from their device, via a browse button or a drag-and-drop zone. |
| **Footer** | The bottom section of a website, home to secondary navigation, legal links, contact details, and social links. |
| **Header** | The top section of a website holding the brand logo, primary nav, and key action buttons. |
| **Help Text** | Small instructional copy near content form fields that guides users on what to enter or what went wrong. |
| **Input** | Single-line text fields for short-form data like names, emails, or search queries. |
| **Input PIN** | Individual character fields purpose-built for numeric codes like PINs, OTPs, and verification entries. |
| **Menu** | Temporary lists of actions or options that appear from a trigger, letting users pick a single command. |
| **Modal** | Full-focus overlay windows for critical information, forms, or workflows that need undivided attention. |
| **Page Controls** | Dots or indicators that show users where they are within a paginated or swipeable set of pages. |
| **Progress Bar** | Horizontal bars that show how far through a task or process the user has come. |
| **Radio** | Select one from a group of mutually exclusive options that work together as a set. |
| **Row** | Horizontal list items showing a label, value, and optional accessory — a staple of settings and list views. |
| **Signup** | Pre-built registration form combining and input and submit action, ready to drop in. |
| **Slider** | Draggable controls for picking a value or range along a track — great for things like volume or price filters. |
| **Spinner** | Animated loading indicators that let users know something is happening behind the scenes. |
| **Tabs** | Selectable labels that switch between content panels, showing one at a time without a page reload. |
| **Table** | Structured grids of rows and columns for displaying, organising, and comparing large sets of data. |
| **Tag** | Small labelled elements for categorising or filtering content — often dismissible or clickable. |
| **Text Area** | Multi-line text fields for longer-form input like messages, comments, or descriptions. |
| **Toast** | Brief, self-dismissing notifications that pop up to confirm an action without interrupting the flow. |
| **Toggle** | On/off switches for settings or features — immediate, satisfying, and instantly understood. |
| **Tooltip** | Small floating labels that appear on hover or focus to give a little extra context about an element. |

### Possible Compositions

The "3,000+" headline counts named variants only. When you factor in every property combination each component supports, the real design space is considerably larger.

| Property type | Count | Notes |
|---|---|---|
| Variant dimensions | across 74 sets | Named variant properties (Type, Size, State, etc.) |
| Variant combinations | 2,496 | Multiplying all variant dimensions per component set |
| Boolean properties | 83 | Show/hide toggles (icons, labels, slots, etc.) |
| **Total compositions** | **45,402** | Variant combos × 2^(boolean count) per component |
| Instance-swap properties | 41 | Icon slot choices — effectively infinite |
| Text properties | 42 | Free-form labels — excluded from count |
